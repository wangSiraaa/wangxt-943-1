## 1. 产品概述

渔港进出港登记系统——面向船长、港口值班员和监管员的全栈协同应用，实现从出港计划提交、多节点审批、放行核验到返港关闭的航次全生命周期闭环管理。系统通过气象预警联动、证书资质校验、泊位容量管控、航线风险评估等业务规则，确保每一次进出港操作合规可追溯，异常情况自动拦截预警。

- 目标用户：渔港船长（出港申请）、港口值班员（核验放行）、监管员（监督审批）
- 核心价值：消除纸质流程、降低安全风险、实现航次全链路数字化监管

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 船长 | 管理员分配账号 | 提交/撤销出港计划、查看航次状态、申报危险品与燃油 |
| 港口值班员 | 管理员分配账号 | 核验船员名单、确认泊位、执行码头放行、处理返港登记 |
| 监管员 | 管理员分配账号 | 抽查审批、查看预警队列、发起临检、关闭航次、查看统计看板 |
| 系统管理员 | 初始超管 | 用户管理、参数配置、系统维护 |

### 2.2 功能模块

1. **值班大屏**：实时展示在港船舶、出港中船舶、预警信息、气象状态、泊位占用
2. **出港计划管理**：计划创建/提交/撤销、多节点审批流、自动规则校验
3. **航次管理**：航次详情、时间线、返港登记、异常返港复核、航次关闭
4. **船舶档案**：船舶基本信息、证书管理、历史航次
5. **船员管理**：船员名单、资质有效期、黑名单
6. **泊位管理**：泊位容量、占用状态、计划分配
7. **预警中心**：气象预警、返港超时、证书过期、航线风险、异常放行告警
8. **统计看板**：进出港统计、航次分析、违规率、审批时效

## 3. 核心流程

### 3.1 出港审批流程

船长提交出港计划 → 系统自动规则校验（证书/船员/泊位/气象/并发航次）→ 校验通过进入审批流 → 港口值班员复核（船员名单确认、泊位确认）→ 监管员抽查（可选，按风险等级决定）→ 码头放行 → 船舶出港 → 返港登记 → 航次关闭

自动打回规则：证书过期、船员黑名单、泊位未确认、气象预警窗口变化、同一船舶存在未关闭航次。

```mermaid

    "船长提交出港计划" --> "系统自动校验"
    "系统自动校验" -->|"证书过期/船员黑名单/泊位未确认/气象预警/并发航次"| "自动打回"
    "系统自动校验" -->|"校验通过"| "值班员复核"
    "值班员复核" -->|"船员名单未确认"| "打回补充"
    "值班员复核" -->|"复核通过"| "监管员抽查判断"
    "监管员抽查判断" -->|"高风险航线/危险品"| "监管员审批"
    "监管员抽查判断" -->|"常规航线"| "码头放行"
    "监管员审批" -->|"不通过"| "打回补充"
    "监管员审批" -->|"通过"| "码头放行"
    "码头放行" --> "船舶出港"
    "船舶出港" --> "返港登记"
    "返港登记" -->|"正常返港"| "航次关闭"
    "返港登记" -->|"异常返港"| "异常复核"
    "异常复核" --> "航次关闭"
```

### 3.2 异常返港复核流程

返港登记时标记异常 → 系统记录偏差原因 → 值班员初步登记 → 监管员复核（检查航线偏离、超时原因、损坏情况）→ 记录复核结论 → 航次关闭

### 3.3 临检流程

监管员发起临检 → 选择目标船舶 → 检查证书/船员/载货 → 记录检查结果 → 违规则暂停航次/合规则继续

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深海蓝 (#0A1628) 为底色，航海蓝 (#1E6091) 为主色，警戒橙 (#E8672C) 为强调色
- **辅助色**：港口绿 (#2D8A56) 表示正常/放行，预警红 (#D93025) 表示危险/打回
- **按钮风格**：圆角矩形（8px），主要操作用实色填充，次要操作用描边
- **字体**：标题用 Noto Sans SC Bold，正文用 Noto Sans SC Regular，数据用 JetBrains Mono
- **布局风格**：左侧导航栏 + 主内容区，大屏模式全屏展示
- **图标风格**：线性图标，2px描边，航海主题（锚、罗盘、波浪等）

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 值班大屏 | 港口态势 | 深色背景、数据卡片、泊位热力图、气象信息条、实时时钟 |
| 值班大屏 | 预警滚动条 | 底部固定横条、橙/红色标签滚动、点击展开详情 |

### 4.3 响应式设计

- 桌面优先设计，最小宽度1280px
- 值班大屏适配1920×1080投屏
- 平板适配768px以上，导航栏折叠为汉堡菜单
- 移动端暂不支持（后续迭代）

### 4.4 动效设计

- 页面加载：卡片依次淡入（stagger 100ms）
- 审批节点：完成时脉冲动画
- 预警条目：新预警滑入+闪烁
- 泊位状态：占用变更时颜色渐变过渡
- 统计图表：数据加载后从0动画绘制
